## 什么是 Git 和 GitHub

### Git
**Git** 是一个分布式版本控制系统，由 Linux 之父 Linus Torvalds 创建。

**通俗理解**：
- 就像游戏的"存档系统"，可以随时保存进度，回到之前的版本
- 可以看到每次修改的历史记录
- 多人协作时不会互相覆盖

**为什么需要 Git**：
```
❌ 没有 Git 的困境：
项目_最终版.zip
项目_最终版2.zip
项目_真的最终版.zip
项目_这次真的是最终版.zip
项目_老板说要改的版本.zip

✅ 使用 Git：
- 所有历史版本都保存在 .git 文件夹
- 随时查看任何时刻的代码
- 清晰的版本记录和说明
```

### GitHub
**GitHub** 是一个基于 Git 的代码托管平台（网站）。

**通俗理解**：
- Git 是工具（像 Word）
- GitHub 是平台（像云盘）
- GitHub = Git + Hub（中心）

**GitHub 的功能**：
1. 托管代码（云存储）
2. 团队协作
3. 项目管理
4. 社交编程（关注、Star、Fork）
5. 开源社区

## 核心概念

### 1. Repository（仓库）
**定义**：存放项目的文件夹

```
我的仓库（Repository）
├── README.md           # 项目说明文档
├── src/               # 源代码目录
│   └── main.py
├── .gitignore         # 忽略文件配置
└── .git/              # Git 版本控制信息（隐藏文件夹）
```

**类型**：
- **本地仓库**（Local Repository）：你电脑上的项目文件夹
- **远程仓库**（Remote Repository）：GitHub 上的项目

### 2. Commit（提交）
**定义**：保存一次项目快照

**理解**：
```
游戏进度：
第1关通关 → 存档点1 (commit 1)
第2关通关 → 存档点2 (commit 2)
获得神器 → 存档点3 (commit 3)

代码版本：
添加登录功能 → commit 1
修复登录 bug → commit 2
添加注册功能 → commit 3
```

**每个 Commit 包含**：
- 修改了什么文件
- 谁修改的
- 什么时候修改的
- 为什么修改（commit message）

### 3. Branch（分支）
**定义**：平行的开发线

**理解**：
```
main 分支（主线剧情）
    │
    ├─→ feature/login（支线任务：开发登录）
    │
    └─→ feature/payment（支线任务：开发支付）
```

**常见分支**：
- `main` / `master`：主分支（稳定版本）
- `develop`：开发分支
- `feature/xxx`：功能分支
- `bugfix/xxx`：修复分支

**为什么需要分支**：
- 不影响主线代码
- 多人并行开发
- 实验新功能
- 随时可以切换或删除

### 4. Merge（合并）
**定义**：把一个分支的改动合并到另一个分支

```
main:     A---B---C-------G
                   \     /
feature:            D---E---F
```

### 5. Pull Request（PR）
**定义**：请求将你的代码合并到主分支

**流程**：
```
1. 你在 feature 分支开发
2. 开发完成后，创建 PR
3. 团队成员 Code Review（代码审查）
4. 讨论、修改
5. 通过后合并到 main 分支
```

### 6. Clone（克隆）
**定义**：把远程仓库下载到本地

```bash
# 从 GitHub 下载项目
git clone https://github.com/username/project.git
```

### 7. Fork（派生）
**定义**：复制别人的仓库到你的账户

```
别人的仓库：github.com/author/project
         ↓ Fork
你的仓库：  github.com/your-name/project
```

**用途**：
- 学习别人的代码
- 贡献开源项目
- 基于别人的项目做自己的修改

### 8. Star（收藏）
**定义**：给项目点赞/收藏

- 表示喜欢或支持这个项目
- 方便以后查找
- 类似社交媒体的"点赞"

### 9. Issue（问题）
**定义**：项目的问题、建议、讨论

**用途**：
- 报告 bug
- 提出新功能建议
- 提问讨论
- 任务追踪

## GitHub 的作用

### 1. 代码托管
- 安全备份代码
- 任何地方都能访问
- 永久免费（公开仓库）

### 2. 版本控制
```
时间线：
2024-01-01: 创建项目 (v0.1)
2024-02-15: 添加用户系统 (v0.2)
2024-03-20: 重构数据库 (v0.3)
2024-04-10: 发布 1.0 (v1.0)

可以随时回到任何版本！
```

### 3. 团队协作
```
团队开发流程：
1. 张三开发登录功能（branch: feature/login）
2. 李四开发支付功能（branch: feature/payment）
3. 王五修复 bug（branch: bugfix/cart）
4. 都开发完成后，通过 PR 合并到主分支
```

### 4. 开源贡献
- 学习优秀代码
- 参与全球开源项目
- 建立个人影响力

### 5. 展示作品
- GitHub 是程序员的简历
- 招聘者会查看你的 GitHub
- Star 数量代表项目认可度

## 注册 GitHub 账户

### 步骤

1. **访问 GitHub**
   - 网址：https://github.com

2. **注册账户**
   - 点击 "Sign up"
   - 输入邮箱、密码、用户名
   - 验证邮箱

3. **完善个人资料**
   ```
   推荐填写：
   - 头像（专业照片或有趣的头像）
   - Bio（简短自我介绍）
   - Location（中国/USTC）
   - Website（个人网站/博客）
   ```

4. **配置 SSH 密钥（可选但推荐）**
   - 避免每次输入密码
   - 更安全的连接方式

```
┌─────────────────────────────────────────────────┐
│          你的电脑（本地环境）                      │
│  ┌──────────────────────────────────────┐      │
│  │  工作区（Working Directory）          │      │
│  │  ├── file1.py                        │      │

│  │  暂存区（Staging Area）               │      │
│  │  准备提交的文件                        │      │

│                    ↓ git commit                 │
│  ┌──────────────────────────────────────┐      │

│  │  .git 文件夹                          │      │

                     ↓ git push

│          GitHub（远程环境）                       │

│  │  远程仓库（Remote Repository）        │      │
│  │  github.com/username/project         │      │
│  └──────────────────────────────────────┘      │
└─────────────────────────────────────────────────┘
```
